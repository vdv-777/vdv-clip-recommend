# 🌐 Нейросетевой сервис поиска и рекомендации модных товаров на основе мультимодальной модели CLIP.

## 📖 О Проекте

Этот проект представляет собой полноценный пайплайн для обучения и использования нейросетевой модели **CLIP (Contrastive Language-Image Pre-training)** для задачи поиска одежды и аксессуаров по текстовому описанию. 

Вместо традиционного поиска по тегам, система понимает семантический смысл запроса (например, *"красная юбка"* или *"кожаная куртка"*) и находит визуально соответствующие товары в базе данных.

### 🧠 Как это работает?
1.  **Понимание текста:** Модель преобразует ваш запрос в векторное представление (эмбеддинг).
2.  **Понимание изображений:** Модель преобразует фотографии товаров в такие же векторы.
3.  **Сопоставление:** Система вычисляет косинусное сходство между вектором запроса и векторами товаров, возвращая наиболее похожие результаты.

### 🌍 Сферы применения
*   🛒 **E-commerce:** Умный поиск в интернет-магазинах одежды.
*   🔍 **Визуальные поисковики:** Поиск аналогов по описанию стиля.
*   🎨 **Рекомендательные системы:** Подбор луков и аксессуаров под описание настроения.
*   📱 **Мобильные приложения:** Голосовой поиск товаров ("найди мне что-то летнее").

## 🤖 Описание Модели и Конфигурация

### Используемая архитектура
*   **Модель:** `openai/clip-vit-base-patch32`
*   **Тип:** Мультимодальная трансформерная модель (Vision Encoder + Text Encoder).
*   **Стратегия:** Fine-Tuning (дообучение) на fashion-датасете.
*   **Размер изображения:** 224x224 пикселей.

### 🔑 Ключевые параметры конфигурации (`CONFIG`)
В начале Notebook (Блок 1) расположен словарь `CONFIG`, позволяющий гибко настраивать процесс:

| Параметр | Значение по умолчанию | Описание |
| :--- | :--- | :--- |
| `model_name` | `openai/clip-vit-base-patch32` | Пресет модели HuggingFace |
| `num_epochs` | `8` | Количество эпох обучения |
| `batch_size_train` | `64` | Размер батча для обучения |
| `learning_rate` | `1e-5` | Скорость обучения оптимизатора |
| `clip_score_target` | `30.0` | Целевой процент схожести (метрика качества) |
| `freeze_image_encoder` | `False` | Разморозка весов визуального энкодера |
| `device` | `cuda` / `cpu` | Автоматическое определение GPU/CPU |

## 📘 Структура Notebook и Назначение Блоков

Проект реализован в виде единого интерактивного Notebook `vdv_clip_recommend.ipynb`, разделенного на логические блоки:

### 🔧 Блок 0: Установка зависимостей
*   **Назначение:** Автоматическая проверка и установка необходимых библиотек (`torch`, `transformers`, `pandas` и др.).
*   **Действие:** Определяет среду (Colab/Kaggle/Local) и устанавливает отсутствующие пакеты. Требует перезагрузки среды после установки torch.

### ⚙️ Блок 1: Конфигурация и Подготовка Данных
*   **Назначение:** Настройка гиперпараметров, путей к файлам и загрузка датасета.
*   **Действие:** Скачивает архив с Kaggle (если нет локально), распаковывает данные, создает папки для чекпоинтов.
*   **Важно:** Здесь редактируются основные настройки обучения.

### 📊 Блок 2: EDA и Диагностика Производительности
*   **Назначение:** Анализ данных (размеры изображений, длина текста) и бенчмарк скорости.
*   **Действие:** Строит графики распределения, оценивает время обработки батча и потребление памяти VRAM. Помогает подобрать оптимальный `batch_size`.

### 🗂️ Блок 3: Датасет и DataLoader
*   **Назначение:** Подготовка данных для обучения.
*   **Действие:** Создает классы `Dataset`, применяет аугментации (для train), разбивает данные на Train/Val/Test выборки.

### 🎓 Блок 4: Обучение Модели
*   **Назначение:** Процесс Fine-Tuning модели CLIP.
*   **Действие:** Запускает цикл обучения, считает.loss, валидирует на тестовой выборке, сохраняет лучший чекпоинт (`clip_fashion_best.pt`) при улучшении метрики CLIP Score.
*   **Результат:** Файлы весов в папке `checkpoints/`.

### 🔍 Блок 5: Поисковая Система (Инференс)
*   **Назначение:** Демонстрация работы обученной модели.
*   **Действие:** Загружает веса, вычисляет эмбеддинги для всей базы изображений (или загружает из кэша), позволяет вводить текстовые запросы и визуализировать найденные товары.
*   **Требование:** Необходимы веса из Блока 4 или скачанные предварительно.

## 🚀 Руководство Пользователя

### 1. Установка зависимостей
Запустите **Блок 0** в Notebook для установки необходимых пакетов и зависимостей.*

### 2. Настройка конфигурации
Перейдите в **Блок 1**. Отредактируйте словарь `CONFIG`, если хотите изменить параметры обучения:
*   Для быстрого теста уменьшите `num_epochs` до 1-2.
*   Если мало памяти GPU, уменьшите `batch_size_train`.
*   Убедитесь, что `FORCE_RETRAIN` стоит в `True`, если вы хотите запустить обучение заново.

### 3. Запуск пайплайна
Выполняйте блоки последовательно сверху вниз:
1.  **Блок 0** → Дождитесь установки пакетов и перезагрузите ядро, если потребуется.
2.  **Блок 1** → Дождитесь загрузки данных.
3.  **Блок 2** → (Опционально) Проверьте метрики данных.
4.  **Блок 3** → Подготовка загрузчиков данных.
5.  **Блок 4** → **Обучение.** Займет от 10 минут до нескольких часов в зависимости от GPU.
6.  **Блок 5** → **Поиск.** Вводите запросы в коде (массив `test_queries`) и смотрите результаты.

### 4. Использование готовых весов
Если вы скачали веса модели и эмбеддинги по ссылкам выше:
1.  Разместите файлы в папках `checkpoints/` и `fashion_data/` согласно структуре.
2.  В **Блоке 1** установите `SKIP_TRAINING = True`.
3.  Запустите **Блок 5** сразу после **Блока 1** и **Блока 3**.
